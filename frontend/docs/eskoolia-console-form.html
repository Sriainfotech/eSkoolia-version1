What's new in this revision

  1. Scan-to-pre-fill bar at the top. Dark, prominent, dismissible. Supports Aadhaar QR, previous school ID, and Eskoolia pre-admission slips. Aadhaar masking is enforced at the UI layer — never show more than the last 4 digits.
  2. Mother tongue, Religion, Nationality added to Identity. Mother tongue is required (state reporting). Religion defaults to "Prefer not to say" and is optional. Nationality defaults to Indian.
  3. New Documents section (05 / 06). Six document upload cards with conditional visibility: Birth certificate (required), Aadhaar (masked), TC + Previous marksheet (only when Admission type = Transfer), Caste/Income certificate (only when Category ≠ Open), Medical info (optional).
  4. RTE Act 2009 compliance banner under the document grid. Explains the 30-day verification rule for EWS/DG admissions and links to the data policy.
  5. Parent consent checkbox before Review. Legally this is the cheapest piece of the form to build and the most expensive one to forget.
  6. Try it: Click "Scan now" in the top bar → "Simulate scan" to see fields auto-fill. Change Admission type to Transfer in the Academic section → TC and Marksheet cards appear. Change Category to any reserved category → Caste certificate card appears.

What changed from the previous revision

  1. Single scrollable form with sticky section nav — not a wizard. Your admission staff will do this 20×/day; wizards add friction for power users. Sections let first-timers jump around.
  2. Auto-generated admission number shown in the field as ADM-2026-0342 with an "Edit" micro-action. Defaults to the right thing; override is one click away.
  3. Drag-drop photo circle replaces the plain file picker. Shows what the ID card will actually look like.
  4. Helper text hides by default, shows on focus. No wall of instructions. The rule: if users need it, they'll tab into the field and see it.
  5. Dependency callout for Class → Section (the purple strip) shows seat availability the moment a class is picked. That's data the admin needs to decide, not a dead "Select class first" message.
  6. Pincode auto-fill has a visible feedback strip with the matched location and a "Clear" link if the lookup is wrong. No more silent magic.
  7. Guardian as a card, not a flat row of fields. Makes "Add another guardian" feel like a real action. Primary guardian is flagged.
  8. Review section at the bottom shows what's been entered and what's still empty — prevents saving a half-empty record.
  9. Sticky footer with progress bar, draft save state, and a primary Save button that is always enabled — clicking it with missing required fields jumps you to the first missing field. Disabled save buttons are user-hostile.
  10. "Mark student disabled" checkbox removed. Status toggle already covers Active/Inactive. One job, one control.
  11. "Multi Subject Assignment" button removed. It doesn't belong on an individual add-student page.

One more thing

The Review section at the bottom isn't decorative — it's the cheapest quality gate you'll ever build. Admission staff make typos. A glance-at-this-before-saving summary catches 80% of "oh wait, wrong class" mistakes that otherwise turn into messy backend cleanup. Cost to build: ~2 hours. Value: every wrong admission you don't have to unpick.
